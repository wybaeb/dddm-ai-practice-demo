# DDDM · Сквозная практика с ИИ-ассистентом — демо-набор

Пристрелочный демонстрационный набор (**v0.1**) для курса «Принятие решений на основе данных и ИИ».

Показывает, как практическое задание курса перекладывается на **ИИ-агента в Telegram**: агент держит
программу в памяти, на каждом шаге выдаёт промпт, собирает промежуточный артефакт (doc/pdf), делает паузу
на ревью слушателя и ведёт дальше — вплоть до финальной презентации-защиты в MARP-шаблоне.

Страница — **интерактивная мини-презентация программы**: 7 этапов, у каждого свой **результат**
(артефакт), плюс живая финмодель (NPV, Tornado, Монте-Карло, корреляции) на вымышленном сквозном
проекте «Пульс» — умные часы с проактивным ИИ-агентом. Это не чек-лист требований к агенту — это
показ того, что слушатель делает на занятиях.

## Демо

GitHub Pages: см. вкладку **Settings → Pages** этого репозитория.
Страница с материалами закрыта паролем (пароль передаётся отдельно); после первого ввода кэшируется в браузере.

🔓 **Ссылку для ИИ-агента (`AGENT.md`) можно давать без пароля.** Парольный гейт (`index.html`)
в **открытой** части содержит ссылку на `AGENT.md` — агент переходит по ней и читает программу из
публичного репозитория, расшифровывать страницу ему не нужно. Под паролем — только презентация для людей.

## Как устроено

| Файл | Назначение |
|---|---|
| **`AGENT.md`** | **Точка входа для агента** — файл, ссылку на который слушатель даёт своему ИИ-агенту (доступна без пароля). Агент читает его первым и начинает вести проект по программе. Содержит приоритет иерархии инструкций. |
| **`PROGRAM.md`** | План занятия: 7 этапов в логике **этап → результат → что делает агент → детальный промпт**. Промпты — рабочие, из материалов курса (предматериалы / М1 / М2). |
| **`ИНСТРУКЦИЯ_ДЛЯ_КОЛЛЕГ.md`** | Как пользоваться: две роли — «ознакомиться» и «протестировать агента». |
| **`CHECKLIST.md`** | **Спецификация агента (Definition of Done) для разработчика** — проверяемые способности (план+память, исследования, коррективы, артефакты, статистика+финмодель, презентация, среда исполнения H). Это инженерный документ; слушателю на странице он не показывается как «требования к агенту». |
| `src/app.html` | Полная интерактивная страница (исходник, без шифрования) |
| `build.py` | Шифрует `src/app.html` паролем (PBKDF2 + AES-256-GCM) → собирает `index.html` |
| `index.html` | То, что отдаётся на GitHub Pages: парольный гейт + зашифрованный payload |

Шифрование на стороне сборки (Python `cryptography`) полностью совместимо с расшифровкой в браузере
(WebCrypto `subtle`): PBKDF2-HMAC-SHA256, 200 000 итераций, AES-GCM.

## Пересборка

```bash
python3 build.py --password 'ВАШ_ПАРОЛЬ'
# или
DDDM_PASSWORD='ВАШ_ПАРОЛЬ' python3 build.py
```

## Что внутри страницы

1. **Запуск агента** — ссылка для агента (без пароля) + готовое сообщение.
2. **Как работает агент** — цикл «промпт → выполнение → результат → пауза на ревью».
3. **Этапы (7 шт.)** — каждый показан как **этап → результат (артефакт)**: дерево метрик, исследование рынка,
   FCM, JTBD-интервью, внешние факторы + записка, EDA+финмодель, презентация-защита. Раскрывается результат,
   что делает агент и ключевой промпт (с кнопкой «копировать»); полные промпты — в `PROGRAM.md`.
4. **Примеры результатов** — дерево метрик и аналитическая записка (факт/интерпретация/гипотеза).
5. **Финмодель** — интерактивные графики: NPV (слайдеры допущений), Tornado, Монте-Карло (P10/P50/P90,
   P(NPV>0)), корреляции.
6. **Презентация-защита** — встроенный просмотрщик слайдов в MARP-шаблоне.

> ⚠️ Все цифры и продукт вымышленные — это демонстрация механики и формата, а не реальная аналитика.
