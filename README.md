# DDDM · Сквозная практика с ИИ-ассистентом — демо-набор

Пристрелочный демонстрационный набор (**v0.1**) для курса «Принятие решений на основе данных и ИИ».

Показывает, как практическое задание курса перекладывается на **ИИ-агента в Telegram**: агент держит
программу в памяти, на каждом шаге выдаёт промпт, собирает промежуточный артефакт (doc/pdf), делает паузу
на ревью слушателя и ведёт дальше — вплоть до финальной презентации-защиты в MARP-шаблоне.

Всё собрано в одну **all-in-one** HTML-страницу с интерактивом (живая финмодель: NPV, Tornado,
Монте-Карло, корреляции) на вымышленном сквозном проекте «Пульс» — умные часы с проактивным ИИ-агентом.

## Демо

GitHub Pages: см. вкладку **Settings → Pages** этого репозитория.
Страница закрыта паролем (пароль передаётся отдельно). После первого ввода пароль кэшируется в браузере.

## Как устроено

| Файл | Назначение |
|---|---|
| **`AGENT.md`** | **Точка входа для агента** — файл, ссылку на который слушатель даёт своему ИИ-агенту. Агент читает его первым и начинает вести проект по программе. |
| **`PROGRAM.md`** | План занятия для агента: 7 шагов с целью, процессом, промптом и выходным артефактом. |
| **`ИНСТРУКЦИЯ_ДЛЯ_КОЛЛЕГ.md`** | Как пользоваться: две роли — «ознакомиться» и «протестировать агента». |
| **`CHECKLIST.md`** | **Чек-лист агента-исполнителя** — спецификация (Definition of Done) для разработчика агента: 49 проверяемых способностей (исследования, план+память, коррективы, артефакты, статистика+финмодель, презентация). Главный документ. |
| `src/app.html` | Полная интерактивная страница (исходник, без шифрования) |
| `build.py` | Шифрует `src/app.html` паролем (PBKDF2 + AES-256-GCM) → собирает `index.html` |
| `index.html` | То, что отдаётся на GitHub Pages: парольный гейт + зашифрованный payload |

Шифрование на стороне сборки (Python `cryptography`) полностью совместимо с расшифровкой в браузере
(WebCrypto `subtle`): PBKDF2-HMAC-SHA256, 200 000 итераций, AES-GCM.

## Пересборка

```bash
python3 build.py --password 'ВАШ_ПАРОЛЬ'
# или
DDDM_PASSWORD='ВАШ_ПАРОЛЬ' python3 build.py
```

## Что внутри страницы

1. **Как работает агент** — цикл «промпт → запуск → артефакт → пауза на ревью».
2. **Программа (7 шагов)** — дерево метрик, исследование рынка, конкурентный анализ (FCM), JTBD-интервью,
   внешние факторы + аналитическая записка, финмодель в Jupyter, презентация-защита. Для каждого шага —
   цель, описание процесса агента и готовый промпт (с кнопкой «копировать»).
3. **Промежуточные артефакты** — дерево метрик и аналитическая записка (факт/интерпретация/гипотеза).
4. **Финмодель** — интерактивные графики: NPV (со слайдерами допущений), Tornado-чувствительность,
   Монте-Карло (P10/P50/P90, P(NPV>0)), корреляционная матрица.
5. **Презентация-защита** — встроенный просмотрщик слайдов в MARP-шаблоне (синий→зелёный градиент).

> ⚠️ Все цифры и продукт вымышленные — это демонстрация механики и формата, а не реальная аналитика.
