# Программа сквозной практики (план занятия для агента)

7 шагов: от бизнес-вопроса до защиты. Для каждого шага — цель, что делает агент, готовый промпт и
выходной артефакт. Демо собрано на вымышленном продукте «Пульс» (умные часы с проактивным ИИ-агентом);
слушатель может взять свой продукт.

После **каждого** шага — пауза на ревью: агент показывает артефакт и ждёт «утверждаем / правим»,
дальше идёт только после согласования. Правила поведения — в [CHECKLIST.md](./CHECKLIST.md).

---

## Шаг 0 — Дерево метрик продукта · Предматериалы
**Артефакт:** дерево метрик (markdown + mermaid + png).
**Цель:** разложить главную метрику результата на управляемые драйверы. База для гипотез и финмодели.

**Что делает агент:**
1. Спрашивает продукт, аудиторию и бизнес-цель квартала.
2. Формирует промпт дерева и просит запустить в ИИ (DeepSeek и т.п.).
3. Классифицирует метрики: опережающие / расчётные / главная.
4. Рендерит дерево в mermaid, сохраняет `.md` + `.png`.
5. Пауза: «проверь связи и формулы — что правим?»

**Промпт:**
```
Ты — продуктовый аналитик. Помоги построить дерево метрик для продукта.
Продукт: умные часы с проактивным ИИ-агентом «Пульс».
Бизнес-цель квартала: рост выручки от подписки за счёт удержания, без роста CAC.
Сделай:
1) Назови ОДНУ главную (запаздывающую) метрику результата.
2) 4–6 опережающих (управляемых) метрик — листья дерева.
3) Расчётные метрики между ними и формулы связи.
Формат: таблица [метрика | тип | формула] + список связей "A → B".
Не выдумывай данные — только структуру и формулы.
```

## Шаг 1 — Бизнес-вопрос и исследование рынка · Модуль 1.2
**Артефакт:** Lean Canvas + лонглист «Проблема × Сегмент» (.doc).
**Цель:** сформулировать 2–3 вопроса исследования и собрать лонглист с приоритезацией сегментов.

**Что делает агент:**
1. Уточняет рынок/домен, фиксирует 2–3 ключевых вопроса.
2. Выдаёт промпт обзора рынка для Perplexity / Deep Research.
3. Просит разделить факты и интерпретации (The Mom Test).
4. Собирает Product Vision (шаблон Мура) и Lean Canvas.
5. Пауза на ревью лонглиста и приоритезации.

**Промпт:**
```
Проведи обзор рынка для продукта «умные часы с проактивным ИИ-агентом».
Дай по пунктам, ОТДЕЛЯЯ факты от оценок:
1) Размер и динамика рынка носимых устройств (со ссылками на источники).
2) Топ-3 сценария использования и боли в каждом.
3) Лонглист «Проблема × Сегмент» (мин. 6 пар) с оценкой остроты 1–5.
Помечай каждое утверждение: [ФАКТ+источник] / [ИНТЕРПРЕТАЦИЯ] / [ГИПОТЕЗА].
Если данных нет — пиши "нет данных", не выдумывай.
```

## Шаг 2 — Конкурентный анализ (FCM) · Модуль 1.3
**Артефакт:** Feature Comparison Matrix + ключевая метрика бенчмаркинга.
**Цель:** отделить «гигиену» от дифференциаторов, выбрать метрику для сравнения.

**Что делает агент:**
1. Просит список 4–6 конкурентов или находит их промптом.
2. Строит Feature Comparison Matrix по функциям.
3. Размечает: гигиена vs дифференциатор.
4. Фиксирует ключевую метрику сравнения и бенчмарки.
5. Пауза: «согласен с дифференциаторами? что добавить?»

**Промпт:**
```
Сравни наш продукт «Пульс» с конкурентами по носимым устройствам.
Построй Feature Comparison Matrix:
- строки — функции (точность пульса, сон, проактивные подсказки, голосовой агент, автономность, цена подписки);
- столбцы — мы + 4 конкурента.
Для каждой функции пометь: [ГИГИЕНА] или [ДИФФЕРЕНЦИАТОР].
В конце: одна ключевая метрика для честного бенчмарка и почему.
```

## Шаг 3 — Интервью и JTBD · Модуль 1.4
**Артефакт:** Jobs to be Done, 4 силы переключения, As-is CJM, гипотезы.
**Цель:** вытащить работы, боли и силы переключения из транскрипций интервью.

**Что делает агент:**
1. Если нет респондентов — генерирует имитацию интервью по промпту (с дисклеймером).
2. Извлекает топ-5 Jobs to be Done.
3. Размечает 4 силы: Push / Pull / Inertia / Anxiety.
4. Строит As-is CJM и 3–5 проверяемых гипотез.
5. Пауза на проверку инсайтов против слов респондента.

**Промпт:**
```
Проанализируй транскрипцию глубинного интервью (вставлю ниже).
Выдели:
1) Топ-5 Jobs to be Done ("когда…, я хочу…, чтобы…").
2) 4 силы переключения: Push, Pull, Inertia, Anxiety — цитатами из текста.
3) As-is CJM: этапы текущего решения боли.
4) 3–5 ПРОВЕРЯЕМЫХ гипотез (с метрикой проверки).
Опирайся только на сказанное. Где домысливаешь — помечай [ИНТЕРПРЕТАЦИЯ].
ТРАНСКРИПЦИЯ: """<агент подставляет транскрипцию>"""
```

## Шаг 4 — Внешние факторы + аналитическая записка · Модуль 1.5–1.6
**Артефакт:** аналитическая записка из 6 разделов (.pdf).
**Цель:** собрать матрицу внешних факторов и свести всё в записку.

**Что делает агент:**
1. Строит матрицу: сезонность, конкуренты, макро, алгоритмы платформ.
2. Подтягивает артефакты шагов 1–3 из памяти.
3. Собирает записку: рынок, конкуренты, JTBD, факторы, гипотезы, вывод.
4. Жёстко разводит ФАКТ / ИНТЕРПРЕТАЦИЯ / ГИПОТЕЗА.
5. Экспорт `.pdf`, пауза на финальное ревью.

**Промпт:**
```
Собери аналитическую записку (1–2 стр.) из наших артефактов. Структура (ровно 6 разделов):
1. Бизнес-вопрос и рынок  2. Конкуренты и дифференциаторы  3. JTBD и силы переключения
4. Матрица внешних факторов  5. Проверяемые гипотезы (с метрикой)  6. Вывод и рекомендация к финмодели
В каждом пункте маркируй [ФАКТ]/[ИНТЕРПРЕТАЦИЯ]/[ГИПОТЕЗА]. Тон — сухой, для принятия решения.
```

## Шаг 5 — Данные, EDA и финмодель · Модуль 2
**Артефакт:** Jupyter-ноутбук (.ipynb) + графики (NPV, Tornado, Монте-Карло, корреляции).
**Цель:** построить дисконтированную модель, тест чувствительности и оценку риска.

**Что делает агент:**
1. Даёт учебный датасет и промпт авто-EDA для Jupyter.
2. Помогает собрать unit-экономику и 3-летнюю модель.
3. Считает NPV, строит Tornado по драйверам.
4. Запускает Монте-Карло и корреляции, объясняет словами.
5. Пауза: «какие допущения меняем перед защитой?»

**Промпт:**
```
Ты — финансовый аналитик в Jupyter. По датасету продаж и подписок:
1) Авто-EDA (пропуски, выбросы, распределения, корреляции) — кодом pandas.
2) 3-летняя модель: выручка устройства + подписка − CAC − постоянные.
3) NPV при ставке 22% и стартовом capex 80 млн ₽.
4) Tornado: ±20% по ARPU, конверсии, retention, CAC, объёму, марже.
5) Монте-Карло (4000 прогонов) → распределение NPV, P10/P50/P90, P(NPV>0).
Каждый график сопровождай выводом в 1 предложение для слайда.
```

## Шаг 6 — Презентация-защита · Финал
**Артефакт:** MARP-дек (.pdf).
**Цель:** собрать 5-минутную защиту по шаблону заказчика из всех артефактов.

**Что делает агент:**
1. Подтягивает все артефакты шагов 0–5 из памяти.
2. Верстает MARP-дек: цели → рынок → JTBD → модель → риск → вывод.
3. Вставляет ключевые графики и одно-предложенные выводы.
4. Экспорт `.pdf`, опц. структура 5-минутного видео.
5. Пауза: финальное ревью перед сдачей.

**Промпт:**
```
Собери защитную презентацию (8 слайдов, MARP) по шаблону заказчика.
Слайды: 1) титул 2) бизнес-вопрос 3) рынок и сегмент 4) дифференциаторы
5) JTBD-инсайт 6) финмодель (NPV) 7) риск (Монте-Карло) 8) вывод и шаги.
Каждый слайд — заголовок + ≤3 буллета + 1 цифра/график.
Стиль — градиент синий→зелёный, крупная типографика. Выдай в формате Markdown для MARP.
```
